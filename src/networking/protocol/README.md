# Игровой протокол обмена сообщениями Forest Kingdoms RPG

Эта директория содержит реализацию игрового протокола обмена сообщениями для P2P мультиплеера в игре Forest Kingdoms RPG.

## Структура директории

- `message_types.gd` - Определения типов игровых сообщений
- `message_structure.gd` - Структура игровых сообщений
- `message_serializer.gd` - Функции сериализации/десериализации сообщений
- `message_handler.gd` - Обработчики различных типов сообщений

## Типы сообщений

### Базовые типы
- `connection` - Подключение к сети
- `disconnect` - Отключение от сети
- `ping` - Проверка соединения
- `pong` - Ответ на проверку соединения

### Типы сообщений о игроках
- `player_join` - Игрок присоединился к игре
- `player_leave` - Игрок покинул игру
- `player_update` - Обновление информации об игроке
- `player_action` - Действие игрока

### Типы сообщений о перемещении
- `movement_start` - Начало перемещения
- `movement_update` - Обновление позиции при перемещении
- `movement_stop` - Окончание перемещения

### Типы сообщений о состоянии игры
- `game_state` - Состояние игры
- `game_event` - Игровое событие
- `game_command` - Команда в игре

### Типы сообщений о мире
- `world_update` - Обновление мира
- `object_update` - Обновление объекта
- `npc_update` - Обновление NPC

### Типы сообщений чата
- `chat_message` - Сообщение в чате
- `chat_private` - Приватное сообщение
- `chat_system` - Системное сообщение

### Типы сообщений о сессии
- `session_create` - Создание сессии
- `session_join` - Присоединение к сессии
- `session_leave` - Покидание сессии
- `session_update` - Обновление сессии

### Типы сообщений о синхронизации
- `sync_request` - Запрос синхронизации
- `sync_response` - Ответ на синхронизацию
- `sync_update` - Обновление синхронизации

### Типы сообщений об ошибках
- `error` - Сообщение об ошибке
- `warning` - Предупреждение

## Структура сообщений

Все сообщения имеют общую структуру:

```gdscript
{
  "type": String,          # Тип сообщения
  "id": String,            # Уникальный идентификатор сообщения
  "timestamp": int,        # Временная метка создания сообщения
  "sender_id": String,     # Идентификатор отправителя
  "recipient_id": String,  # Идентификатор получателя (если применимо)
  "data": Dictionary,       # Данные сообщения
  "priority": int,         # Приоритет сообщения
  "ttl": int               # Время жизни сообщения
}
```

## Сериализация сообщений

Протокол поддерживает два типа сериализации:

1. **JSON** - человекочитаемый формат, удобный для отладки
2. **Бинарный** - компактный и быстрый формат для производительности

Выбор типа сериализации зависит от типа сообщения:
- Для сообщений о перемещении и синхронизации используется бинарная сериализация
- Для остальных сообщений используется JSON

## Использование

### Инициализация обработчиков

```gdscript
const MessageHandler = preload("res://src/networking/protocol/message_handler.gd")

var message_handler = MessageHandler.new()
message_handler.initialize(node_os_adapter, existing_message_handler)
```

### Отправка сообщений

```gdscript
# Отправка сообщения о подключении игрока
message_handler.send_player_join_message(
    peer_id, 
    "player1", 
    "Alice", 
    {"level": 1, "class": "warrior"}
)

# Отправка сообщения о перемещении
message_handler.send_movement_update_message(
    peer_id, 
    "player1", 
    Vector3(10, 0, 20), 
    Vector3(1, 0, 0)
)

# Отправка сообщения в чат
message_handler.send_chat_message(
    peer_id, 
    "Alice", 
    "Привет, мир!", 
    "global"
)
```

### Обработка входящих сообщений

Обработчики автоматически подключаются к сигналам существующего MessageHandler и обрабатывают входящие сообщения в зависимости от их типа.

## Сигналы

Обработчик сообщений эмитирует следующие сигналы:

- `player_joined(player_id, player_data)` - Игрок присоединился
- `player_left(player_id)` - Игрок покинул игру
- `player_moved(player_id, position, velocity)` - Игрок переместился
- `chat_message_received(sender_name, message_text, channel)` - Получено сообщение в чате
- `game_state_updated(state_data)` - Обновлено состояние игры
- `error_occurred(error_code, error_message)` - Произошла ошибка

## Интеграция

Протокол интегрирован с существующими сетевыми компонентами:

- **NodeOSAdapter** - для взаимодействия с P2P нодой
- **MessageHandler** - для отправки и получения сообщений
- **ConnectionManager** - для управления соединениями

## Производительность

Для оптимизации производительности:

1. Используется бинарная сериализация для часто передаваемых сообщений
2. Реализованы механизмы проверки времени жизни сообщений
3. Предусмотрены приоритеты сообщений для управления очередью

## Расширяемость

Протокол легко расширяем:

1. Добавление новых типов сообщений в `message_types.gd`
2. Создание новых структур сообщений в `message_structure.gd`
3. Добавление обработчиков в `message_handler.gd`
4. Реализация специфической сериализации в `message_serializer.gd`
