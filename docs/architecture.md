# Архитектура проекта Forest Kingdoms RPG

## Обзор

Forest Kingdoms RPG - это 3D RPG с открытым миром, разрабатываемая с использованием Godot Engine и децентрализованной P2P архитектуры. Проект следует модульной архитектуре с четким разделением ответственности между компонентами.

## Общая архитектура

### Уровни архитектуры

```
┌─────────────────────────────────────┐
│           Пользовательский          │
│             интерфейс               │
├─────────────────────────────────────┤
│         Игровая логика             │
├─────────────────────────────────────┤
│         Системы и менеджеры        │
├─────────────────────────────────────┤
│         P2P интеграция             │
├─────────────────────────────────────┤
│         Базовые компоненты         │
└─────────────────────────────────────┘
```

### Основные компоненты

1. **Игровой движок** - Godot Engine 4.0
2. **P2P фреймворк** - Децентрализованная сеть
3. **Система процедурной генерации** - Генерация мира
4. **Система повреждений** - Реалистичные повреждения персонажа
5. **Система квестов** - Квестовая система
6. **Экономическая система** - Торговля и ресурсы

## Архитектура игрового движка

### Структура проекта

```
src/
├── actors/                 # Актеры и персонажи
├── ai/                      # Системы искусственного интеллекта
├── combat/                  # Боевые системы
├── environment/            # Окружающая среда
├── game/                    # Основная игровая логика
├── gui/                     # Пользовательский интерфейс
├── items/                   # Предметы и инвентарь
├── networking/              # Сетевые компоненты
├── p2p/                     # P2P интеграция
├── physics/                 # Физические системы
├── procedural/              # Процедурная генерация
├── quests/                  # Система квестов
├── save/                     # Система сохранения
├── sound/                   # Звуковая система
└── utils/                   # Вспомогательные утилиты
```

## P2P архитектура

### Компоненты P2P

1. **IdentityLayer** - Идентификация узлов
2. **StateSyncLayer** - Синхронизация состояния
3. **DiscoveryLayer** - Обнаружение узлов
4. **SecurityLayer** - Безопасность
5. **NetworkLayer** - Сетевые протоколы

### Интеграция с Godot

```
P2PAdapter (JavaScript)
├── IdentityManager
├── StateManager
├── NetworkManager
└── SecurityManager

GodotP2P (GDScript)
├── P2PManager
├── CRDTManager
└── SignalManager
```

### Типы CRDT

1. **LWWRegister** - Последнее значение побеждает
2. **ORSet** - Наблюдаемое множество
3. **MVRegister** - Множественные значения
4. **Counter** - Распределенный счетчик
