# P2P Фреймворк

## Обзор

P2P фреймворк - это универсальный фреймворк для создания децентрализованных мультиплеерных приложений. Он предоставляет набор инструментов для обнаружения узлов, установления соединений, обмена сообщениями и управления сессиями в P2P сети.

## Архитектура

Фреймворк состоит из следующих основных компонентов:

### 1. P2PFramework (Основной класс)
Центральный класс, координирующий работу всех компонентов фреймворка.

### 2. DiscoveryManager
Отвечает за обнаружение других узлов в сети и доступных сессий.

### 3. ConnectionManager
Управляет сетевыми соединениями между узлами.

### 4. MessageHandler
Обрабатывает отправку и получение сообщений между узлами.

### 5. SessionManager
Управляет игровыми сессиями, включая создание, подключение и управление игроками.

### 6. ErrorHandler
Обрабатывает ошибки и обеспечивает механизмы восстановления.

## Установка и настройка

### Добавление фреймворка в проект

1. Скопируйте директорию `src/networking` в свой проект
2. Добавьте `P2PFramework.gd` в автозагрузку проекта:
   - Откройте Project Settings > Autoload
   - Добавьте `res://src/networking/P2PFramework.gd` как автозагрузочный скрипт
   - Назовите его `P2PFramework`

### Базовая инициализация

```gdscript
# Получение экземпляра фреймворка
var p2p = P2PFramework.get_instance()

# Установка информации о локальном узле
p2p.session_manager.set_local_peer_info("PlayerName")

# Запуск фреймворка
p2p.start()
```

## Основные функции

### Создание сессии

```gdscript
# Создание новой игровой сессии
var session_id = p2p.create_session("Моя игра", 4)
if session_id != -1:
    print("Сессия создана с ID: ", session_id)
```

### Подключение к сессии

```gdscript
# Подключение к существующей сессии
p2p.join_session(session_id)
```

### Отправка сообщений

```gdscript
# Отправка сообщения другому узлу
var message_data = {
    "text": "Привет, мир!",
    "timestamp": Time.get_ticks_msec()
}

p2p.send_message(peer_id, "chat", message_data)
```

### Получение сообщений

Подключитесь к сигналу `message_received`:

```gdscript
p2p.connect("message_received", self, "_on_message_received")

func _on_message_received(message_data):
    match message_data.type:
        "chat":
            print("Чат сообщение: ", message_data.data.text)
        "game_data":
            # Обработка игровых данных
            pass
```

## Типы сообщений

Фреймворк поддерживает следующие стандартные типы сообщений:

- `handshake` - Обмен информацией при установлении соединения
- `chat` - Чат-сообщения
- `game_data` - Игровые данные
- `session_info` - Информация о сессии
- `peer_info` - Информация об узле
- `disconnect` - Сообщение о разрыве соединения

## Обработка ошибок

Фреймворк предоставляет систему обработки ошибок:

```gdscript
p2p.connect("error_occurred", self, "_on_error_occurred")

func _on_error_occurred(error_code, error_message):
    print("Ошибка: ", error_message)
    # Попытка автоматического восстановления
    p2p.error_handler.attempt_error_recovery(error_code)
```

## Расширение функциональности

### Добавление новых типов сообщений

1. Определите тип сообщения в `MessageHandler.gd`
2. Добавьте обработчик в функцию `register_message_handlers()`
3. Реализуйте логику обработки сообщения

### Настройка параметров

Основные параметры фреймворка можно настроить:

```gdscript
# Интервал обнаружения узлов (в секундах)
p2p.discovery_manager.discovery_interval = 10.0

# Максимальное количество соединений
p2p.connection_manager.max_connections = 64

# Порог критической ошибки
p2p.error_handler.critical_error_threshold = 10
```

## Лучшие практики

1. **Инициализация**: Всегда инициализируйте фреймворк при запуске приложения
2. **Обработка ошибок**: Подключайтесь к сигналам ошибок и реализуйте механизмы восстановления
3. **Очистка ресурсов**: Останавливайте фреймворк при завершении приложения
4. **Безопасность**: Проверяйте все входящие сообщения на корректность
5. **Производительность**: Используйте очереди сообщений для больших объемов данных

## API Reference

### P2PFramework

#### Методы
- `initialize()` - Инициализация фреймворка
- `start()` - Запуск фреймворка
- `stop()` - Остановка фреймворка
- `send_message(peer_id, message_type, data)` - Отправка сообщения
- `create_session(session_name, max_players)` - Создание сессии
- `join_session(session_id)` - Подключение к сессии
- `get_available_sessions()` - Получение списка доступных сессий

#### Сигналы
- `peer_connected(peer_id)` - Узел подключен
- `peer_disconnected(peer_id)` - Узел отключен
- `message_received(message_data)` - Получено сообщение
- `error_occurred(error_code, error_message)` - Произошла ошибка

### SessionManager

#### Методы
- `create_session(session_name, max_players)` - Создание сессии
- `join_session(session_id)` - Подключение к сессии
- `leave_session(session_id)` - Покидание сессии
- `get_session_info(session_id)` - Получение информации о сессии
- `update_game_state(session_id, game_state, custom_data)` - Обновление состояния игры

## Устранение неполадок

### Распространенные проблемы

1. **Не удается обнаружить другие узлы**
   - Проверьте настройки сети и брандмауэра
   - Убедитесь, что фреймворк запущен

2. **Соединение разрывается**
   - Проверьте стабильность сети
   - Увеличьте таймауты соединений

3. **Сообщения не доставляются**
   - Проверьте правильность сериализации данных
   - Убедитесь, что узлы подключены

### Логирование

Фреймворк предоставляет подробное логирование для диагностики проблем. Проверяйте консоль на наличие сообщений об ошибках.
