# Интеграция P2P компонента в Forest Kingdoms RPG

## Обзор

Этот документ описывает интеграцию децентрализованного P2P компонента в игру Forest Kingdoms RPG. Интеграция позволяет игре использовать продвинутые возможности синхронизации состояния и идентификации узлов через CRDT и DID.

## Архитектура

### Компоненты P2P

1. **IdentityLayer** - слой идентификации, реализующий DID документы на основе ключей и привязку к Nostr публичным ключам
2. **StateSyncLayer** - слой синхронизации состояния, использующий CRDT для обеспечения консистентности данных между узлами
3. **CRDT реализации** - конкретные реализации CRDT, такие как LWWRegister и ORSet

### Адаптер P2P

В игре используется адаптер P2P, который интегрирует JavaScript компоненты с игровым движком Godot. Адаптер предоставляет следующие функции:

1. Инициализация идентификационного слоя и генерация Nostr ключей
2. Создание и управление DID документами узлов
3. Создание и управление CRDT для синхронизации игрового состояния
4. Сериализация и десериализация состояния для передачи между узлами

## Интеграция с игровым фреймворком

### P2PFramework

Основной класс P2P фреймворка игры был расширен для поддержки нового функционала:

1. Добавлен компонент `p2p_adapter` для взаимодействия с P2P компонентами
2. Добавлены методы для создания и управления CRDT игрового состояния
3. Интегрированы сигналы для обработки событий синхронизации

### Использование в игре

Для использования P2P функциональности в игре, разработчики могут:

1. Создавать CRDT для различных аспектов игрового состояния (позиции игроков, состояние объектов и т.д.)
2. Синхронизировать изменения состояния между узлами через существующую сетевую инфраструктуру
3. Использовать DID для безопасной идентификации узлов в сети

## Примеры использования

### Создание CRDT для позиции игрока

```javascript
// Создание LWWRegister для хранения позиции игрока
const playerPositionCRDT = p2pAdapter.createGameStateCRDT(
  'player-position-' + playerId,
  'LWWRegister',
  {x: 0, y: 0}
);

// Обновление позиции игрока
playerPositionCRDT.set({x: 10, y: 20}, nodeId);
```

### Синхронизация состояния между узлами

```javascript
// Сериализация состояния для отправки другим узлам
const serializedState = p2pAdapter.serializeGameState();

// Отправка состояния через существующую сетевую инфраструктуру
networkManager.sendData(peerId, serializedState);

// На принимающем узле - десериализация и слияние состояния
p2pAdapter.deserializeGameState(receivedState);
```

## Безопасность

Интеграция использует следующие механизмы безопасности:

1. **DID документы** для идентификации узлов
2. **Nostr ключи** для криптографической привязки идентификаторов
3. **JWT подписи** для подтверждения связей между DID и Nostr ключами

## Расширение функциональности

Для расширения P2P функциональности можно:

1. Добавить новые типы CRDT для специфических аспектов игрового состояния
2. Реализовать дополнительные методы синхронизации для оптимизации передачи данных
3. Интегрировать более продвинутые механизмы обнаружения узлов
4. Добавить поддержку офлайн режима с последующей синхронизацией
