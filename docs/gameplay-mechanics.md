## Сетевая архитектура

### Общая архитектура
Forest Kingdoms RPG использует децентрализованную P2P архитектуру для мультиплеерного режима. Это позволяет:
- Игрокам напрямую подключаться друг к другу без центрального сервера
- Распределять нагрузку между всеми участниками сессии
- Обеспечивать отказоустойчивость сети

### Компоненты сетевой системы

#### P2P Фреймворк
Универсальный фреймворк для создания децентрализованных мультиплеерных приложений, включающий:
- **DiscoveryManager** - Обнаружение узлов и сессий
- **ConnectionManager** - Управление сетевыми соединениями
- **MessageHandler** - Обработка обмена сообщениями
- **SessionManager** - Управление игровыми сессиями
- **ErrorHandler** - Обработка ошибок и восстановление соединений

#### Слои сетевой архитектуры
1. **Транспортный слой** - Обеспечивает надежную передачу данных между узлами
2. **Слой сессий** - Управляет игровыми сессиями и состоянием игроков
3. **Слой сообщений** - Обрабатывает различные типы игровых сообщений
4. **Слой игровой логики** - Синхронизирует состояние игры между всеми участниками

### Типы сетевых взаимодействий

#### Обнаружение и подключение
- Автоматическое обнаружение доступных сессий в локальной сети
- Глобальное обнаружение через координационный сервер
- Поддержка прямого подключения по IP адресу

#### Синхронизация игрового состояния
- Передача позиций персонажей и объектов
- Синхронизация состояния игрового мира
- Обработка входов игроков (input prediction и reconciliation)

#### Обмен сообщениями
- Чат между игроками
- Передача игровых команд
- Уведомления о событиях в игре

### Режимы игры

#### Кооперативный режим

### Обработка задержек и потерь пакетов
