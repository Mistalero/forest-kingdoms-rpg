# Текстовый MUD режим для Forest Kingdoms RPG

## Обзор

Текстовый MUD (Multi-User Dungeon) режим позволяет игрокам взаимодействовать с игрой Forest Kingdoms RPG через текстовый интерфейс, что обеспечивает доступность для игроков с низкой производительностью и возможность игры через терминал.

## Архитектура MUD режима

### Основные компоненты

1. **TextInterface** - текстовый интерфейс для взаимодействия с игроком
2. **TextCommandProcessor** - обработчик текстовых команд
3. **TextWorldRenderer** - текстовое представление игрового мира
4. **TextCombatSystem** - текстовая система боя
5. **TextInventorySystem** - текстовая система инвентаря
6. **TextQuestSystem** - текстовая система квестов
7. **TextSocialSystem** - текстовая система социальных взаимодействий

### Интеграция с основной игрой

MUD режим интегрируется с основной игрой через следующие компоненты:

1. **GameStateManager** - общий менеджер состояния игры
2. **P2PNetworkLayer** - общий сетевой слой
3. **SaveSystem** - общий механизм сохранения
4. **QuestSystem** - общий механизм квестов

## Команды MUD режима

### Навигация
- `go <direction>` - перемещение в указанном направлении
- `look` - осмотреть текущую локацию
- `examine <object>` - осмотреть объект
- `map` - показать карту окрестностей

### Взаимодействие
- `take <item>` - взять предмет
- `drop <item>` - бросить предмет
- `use <item>` - использовать предмет
- `talk <npc>` - поговорить с NPC

### Бой
- `attack <target>` - атаковать цель
- `cast <spell>` - произнести заклинание
- `flee` - попытаться убежать

### Инвентарь
- `inventory` или `i` - показать инвентарь
- `equip <item>` - экипировать предмет
- `unequip <item>` - снять предмет

### Квесты
- `quests` - показать список квестов
- `quest <id>` - показать детали квеста

### Социальные функции
- `say <message>` - сказать что-то
- `whisper <player> <message>` - шепнуть игроку
- `emote <action>` - выполнить эмоцию

### Системные команды
- `help` - показать справку
- `save` - сохранить игру
- `quit` - выйти из игры
- `settings` - настройки игры

## Текстовое представление мира

### Локации
Каждая локация описывается текстом, включающим:
- Название локации
- Описание окружения
- Список видимых объектов
- Список видимых NPC
- Возможные направления движения

Пример:
```
Лесная тропа
Вы находитесь на узкой тропе, ведущей через густой лес. Солнечный свет едва пробивается сквозь листву.
На севере тропа ведет к руинам старого храма.
На юге тропа ведет к деревне.
Видны: старый меч, труп ворона
NPC: лесной стражник
```

### Объекты
Каждый объект имеет текстовое описание и параметры:
- Название
- Описание
- Тип (оружие, броня, расходуемый предмет и т.д.)
- Параметры (урон, защита, вес и т.д.)

### NPC
Каждый NPC имеет:
- Имя
- Описание
- Диалоги
- Задачи
- Инвентарь

## Система боя

### Основные принципы
- Пошаговая система боя
- Описание действий текстом
- Возможность использования заклинаний и способностей

### Пример боя:
```
Вы сталкиваетесь с лесным волком!
Лесной волк: Здоровье 20/20
Вы: Здоровье 50/50

> attack wolf
Вы наносите волку 8 урона!
Лесной волк наносит вам 3 урона!
Лесной волк: Здоровье 12/20
Вы: Здоровье 47/50
```

## Сохранение и загрузка

MUD режим использует тот же формат сохранения, что и основная игра, что позволяет:
- Загружать сохранения, созданные в 3D режиме
- Продолжать игру в 3D режиме после игры в MUD режиме
- Синхронизировать прогресс между режимами

## P2P интеграция

MUD режим полностью интегрирован с P2P архитектурой игры:
- Использует те же протоколы обмена сообщениями
- Поддерживает те же механизмы обнаружения узлов
- Использует те же механизмы синхронизации состояния (CRDT)
- Поддерживает те же механизмы безопасности (DID)

## Настройки

MUD режим поддерживает следующие настройки:
- Цветной/черно-белый вывод
- Длина описаний (краткие/полные)
- Формат отображения параметров
- Горячие клавиши
- Автосохранение

## Разработка

### Структура кода
```
src/mud/
├── interface/              # Текстовый интерфейс
├── commands/                # Обработчики команд
├── world/                   # Представление мира
├── combat/                  # Система боя
├── inventory/               # Система инвентаря
├── quests/                  # Система квестов
├── social/                  # Социальные функции
└── utils/                   # Вспомогательные утилиты
```

### API
MUD режим предоставляет следующие API:
- `MUDInterface` - основной интерфейс
- `MUDCommandProcessor` - обработчик команд
- `MUDWorldRenderer` - рендерер мира
- `MUDCombatSystem` - система боя
- `MUDInventorySystem` - система инвентаря
